National Cadaster Database Architecture

A unified spatial database categorized into three "realms"

In-Process Data

Conversion & Staging

Contains the conversion of the cadastral plan features. Used to update the Parcel Fabric data once the plan is approved.

  • Table Definition Cadastral plan (process) boundaries are defined by the complete outer boundary of the parcels - either the input parcels or the resulting parcels, depending on the process type.
    This table stores general information about the cadastral plan. Each plan has a unique name.
    CadasterProcessBorders
  • Table Definition These 2D parcels are derived from the plan drawing and stored as staged features until the corresponding cadastral plan is approved by the cadastral reviewer.
    InProcessParcels2D
  • Table Definition These 2D border points are derived from the plan drawing and stored as staged features until the corresponding cadastral plan is approved by the cadastral reviewer. The points in this table does not contain z-values.
    InProcessBorderPoints
  • Table Definition These fronts are the lines of the in-process 2D parcels that derived from the plan drawing and stored as staged features until the corresponding cadastral plan is approved by the cadastral reviewer.
    InProcessFronts
  • Table Definition These 3D parcels are derived from the (3D) plan drawing and stored as staged features until the corresponding cadastral plan is approved by the cadastral reviewer.
    InProcessParcels3D
  • Table Definition These are the projected 2D representations of the in-process 3D parcels. each projection references its corresponding in-process 3D parcel.
    InProcessProjectedParcels3D
  • Table Definition These substractions are 3D features derived from the (3D) plan drawing and stored as staged features until the corresponding cadastral plan is approved by the cadastral reviewer.
    InProcessSubstractions
  • Table Definition These are the projected 2D representations of the in-process substractions. each projection references its corresponding in-process substraction.
    InProcessProjectedSubstractions
  • Table Definition These 3D border points are derived from the (3D) plan drawing and stored as staged features until the corresponding cadastral plan is approved by the cadastral reviewer. The points in this table contains z-values.
    InProcessBorderPoints3D

Management

Validation & Logs

Used for maintenance, validation, and history tracking.

  • Table Definition Stores log information for ArcGIS tasks executed in the past. The table includes execution dates, user details, and task success status. Its primary use is for KPI tracking and performance analysis.
    TasksHistory
  • Table Definition Stores status changes of plans (processes) throughout their lifecycle. Each status update is associated with a specific date and time.
    CPBStatusAndDates
  • Table Definition A set of four standard tables managed and controlled by ArcGIS Attribute Rules: GDB_ValidationLineErrors, GDB_ValidationObjectErrors, GDB_ValidationPointErrors, and GDB_ValidationPolygonErrors.
    Attribute Rules Validations
  • Table Definition Stores the cadastral actions (Divide, Merge, Tranffer, Create, Cancel) applied to parcels within each plan.
    SequenceActions
